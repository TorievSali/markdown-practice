# Блок-схема алгоритма решения квадратного уравнения

**Описание алгоритма:**  
Алгоритм принимает три коэффициента квадратного уравнения \( ax^2 + bx + c = 0 \), проверяет, является ли уравнение квадратным, вычисляет дискриминант и на основе его значения определяет количество корней (два, один или ни одного). Результат выводится на экран.

---

## Диаграмма

```mermaid
flowchart TD
    Start([Начало]) --> Input[/"Ввод коэффициентов a, b, c"/]
    Input --> CheckA{"a == 0?"}
    CheckA -- "Да" --> NotQuad[Вывод: «Не квадратное уравнение»]
    NotQuad --> Stop([Конец])
    CheckA -- "Нет" --> CalcD["Вычислить дискриминант: D = b² - 4ac"]
    CalcD --> CheckD{"D > 0?"}
    CheckD -- "Да" --> TwoRoots["x₁ = (-b + √D) / (2a); x₂ = (-b - √D) / (2a)"]
    CheckD -- "Нет" --> CheckZero{"D == 0?"}
    CheckZero -- "Да" --> OneRoot["x = -b / (2a)"]
    CheckZero -- "Нет" --> NoRoot[Вывод: «Действительных корней нет»]
    TwoRoots --> Output[/"Вывод x₁, x₂"/]
    OneRoot --> Output
    NoRoot --> Stop2([Конец])
    Output --> Stop2

---

# Ответы на контрольные вопросы

## 1. Что такое Mermaid и для чего он используется?

Mermaid — это специальный язык разметки, который позволяет создавать диаграммы и блок-схемы с помощью текста.  
Он используется для визуального представления алгоритмов, процессов и различных структур в Markdown-документах.

## 2. Как вставить диаграмму в Markdown-документ?

Для вставки диаграммы используется блок кода с указанием языка `mermaid`.

Пример:

```mermaid
flowchart TD
A --> B
```

## 3. Какие типы узлов доступны в блок-схемах Mermaid?

В Mermaid можно использовать разные типы узлов:

- прямоугольник — процесс;
- ромб — условие;
- овал — начало или конец;
- параллелограмм — ввод и вывод данных;
- цилиндр — база данных;
- круг — соединение элементов.

## 4. Чем отличаются стрелки `-->` и `-- текст -->`?

Стрелка `-->` показывает обычный переход между элементами схемы.  
Стрелка `-- текст -->` позволяет добавить подпись к переходу, например «Да» или «Нет».

## 5. Как изменить ориентацию диаграммы с вертикальной на горизонтальную?

Для этого нужно изменить направление графа.

Например:

- `TD` — сверху вниз;
- `LR` — слева направо.

Пример горизонтальной схемы:

```mermaid
flowchart LR
A --> B
```

## 6. Зачем нужны подграфы (subgraph)?

Подграфы используются для объединения нескольких элементов схемы в отдельную группу.  
Это помогает сделать большие диаграммы более понятными и удобными для чтения.

## 7. Какие символы нельзя использовать в идентификаторах узлов?

В идентификаторах узлов нельзя использовать пробелы и некоторые специальные символы.  
Обычно применяются буквы и цифры.

## 8. Почему важно указывать начальный и конечный узлы?

Начальный и конечный узлы показывают, где алгоритм начинается и где заканчивается.  
Это делает блок-схему более понятной и логически завершённой.
